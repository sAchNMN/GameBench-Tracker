# 开发路线

## 首批任务

### 批次 A：Phase 0 文档

本次只允许执行批次 A。

- T001 创建需求与边界文档。
- T002 创建数据库设计文档。
- T003 创建 API 契约文档。
- T004 创建时间、性能、退出规范。
- T005 创建测试计划和任务路线。

### 批次 B：项目基础

批次 B 本次不得执行。

- T006 初始化后端空项目。
- T007 建立统一响应结构。
- T008 建立全局异常和校验骨架。
- T009 接入 SQLite 基础。

### 批次 C：游戏后端

批次 C 本次不得执行。

- T010 创建 game 表和实体。
- T011 游戏新增接口和测试。
- T012 游戏列表接口和测试。
- T013 游戏详情接口和测试。
- T014 游戏编辑接口和测试。


## 每个任务必须包含

1. 任务目标。
2. 当前前置状态。
3. 允许修改的文件或目录。
4. 禁止修改的文件或目录。
5. 需要新增的文件。
6. 需要修改的文件。
7. 功能要求。
8. 数据库影响。
9. API 影响。
10. 前端影响。
11. 验收步骤。
12. 自动化测试要求。
13. Git 检查要求。
14. 文档更新要求。
15. 完成后的报告格式。
